# Web Content Memory Structure

## Directory Organization

```
web-content/
├── templates/           # Content templates and patterns
├── published/          # Live content
├── drafts/            # Work in progress
├── archive/           # Historical content
├── analytics/         # Performance data
│   ├── daily/
│   ├── weekly/
│   └── monthly/
└── campaigns/         # Marketing campaigns
    ├── active/
    ├── planned/
    └── completed/
```

## File Naming Conventions

1. Content Files: `YYYY-MM-DD_[type]_[title].md`
   Example: `2024-03-04_blog_cannabis-safety.md`

2. Analytics Files: `YYYY-MM-DD_[metric].json`
   Example: `2024-03-04_website-traffic.json`

3. Campaign Files: `[campaign-name]_[YYYY-MM].md`
   Example: `spring-promo_2024-03.md`

## Content Guidelines

### Templates
- Blog post structure
- Product descriptions
- Social media formats
- Email templates
- Landing pages

### Published Content
- Final approved versions
- Live on website
- Active campaigns
- Current promotions

### Drafts
- Work in progress
- Pending review
- Scheduled content
- Future campaigns

### Analytics
- Traffic metrics
- Engagement data
- Conversion rates
- Campaign performance
- SEO rankings

## Archival Policy
- Archive after content retirement
- Keep analytics for 24 months
- Maintain campaign history
- Document performance data
