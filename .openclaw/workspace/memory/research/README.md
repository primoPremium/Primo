# Research Memory Structure

## Directory Organization

```
memory/research/
├── daily/                 # Daily research findings and updates
│   └── YYYY/             # Year-based organization
│       └── MM-DD.md      # Daily entries
│
├── competitors/           # Competitor analysis and tracking
│   └── YYYY-MM/          # Monthly organization
│       ├── eaze.md       # Individual competitor files
│       ├── grassdoor.md
│       └── amuse.md
│
├── trends/               # Market and industry trends
│   └── YYYY/            # Year-based organization
│       ├── q1.md        # Quarterly trend reports
│       ├── q2.md
│       ├── q3.md
│       └── q4.md
│
├── reports/              # Formal research reports
│   └── YYYY/            # Year-based organization
│       ├── weekly/      # Weekly summaries
│       ├── monthly/     # Monthly analysis
│       └── quarterly/   # Quarterly deep dives
│
└── archive/             # Historical data over 1 year old
```

## File Naming Conventions

1. Daily Files: `YYYY/MM-DD.md`
   Example: `daily/2024/03-01.md`

2. Competitor Files: `YYYY-MM/[competitor-name].md`
   Example: `competitors/2024-03/eaze.md`

3. Trend Reports: `YYYY/[period].md`
   Example: `trends/2024/q1.md`

4. Research Reports: `YYYY/[type]/[name]-[date].md`
   Example: `reports/2024/monthly/market-analysis-2024-03.md`

## Content Guidelines

### Daily Entries
- Date and time stamps for all updates
- Source links
- Key findings
- Action items
- Market changes

### Competitor Files
- Monthly snapshots
- Price changes
- Product updates
- Marketing campaigns
- Service changes

### Trend Reports
- Market data
- Consumer behavior
- Technology adoption
- Regulatory updates
- Industry forecasts

### Research Reports
- Executive summary
- Detailed analysis
- Data visualizations
- Recommendations
- Source documentation

## Archival Policy
- Files over 1 year old move to archive/
- Maintain folder structure in archive
- Include context summary in archived files
